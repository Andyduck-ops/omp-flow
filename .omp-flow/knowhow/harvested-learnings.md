# Harvested Learnings & Recipes

- Gotcha: Always check for typescript type safety in OMP extensions.

- Gotcha: OMP extension tools bypass `AgentDefinition.tools` whitelists unless registered with `defaultInactive: true`; non-default-inactive extension tools are always included by the runtime, so per-agent isolation must start by removing that backdoor.
- Recipe: Use the pre-made toolbelt pattern for sensitive OMP tools: register the tool as `defaultInactive`, put it only in the intended `.omp/agents/{role}.md` `tools` whitelist, and keep execute-time checks as defense in depth rather than the primary authorization boundary.
- Gotcha: OMP extension tool `execute()` receives `ExtensionContext`, not `AgentToolContext`; it has no trusted `agentId`, `taskDepth`, role, or row binding, so dynamic role authorization inside extension-tool execute is not implementable without an upstream runtime identity change.
- Recipe: Use `createRequire(import.meta.url)` plus a lazy `require()` inside the function that needs an OMP runtime module. This lets `tsc` use ambient declarations while avoiding `tsx` module-load failures for host-provided packages that are absent from workspace `node_modules`.
- Gotcha: `ABSOLUTE_NO_WRITE` control-plane protections can self-block trusted reviewers; a grill/reviewer asked to write `GRILL.json` under `.task/` may be blocked by the same `onToolCall` guard that protects task state, so review artifacts need a host-managed verdict path or an explicitly allowed output location.
- Gotcha: LiteLLM gateway parameter limits can be exceeded by a large Main-agent toolbelt; around 35 tools with complex schemas can breach a 100-parameter gateway cap and produce 400 errors.
- Recipe: Use a lite orchestrator Main session: keep only control-plane and coordination tools, remove implementation tools such as `bash`, `lsp`, `ast_grep`, and `ast_edit`, and expose `omp_flow_dispatch` additively only to the captured Main session.
- Gotcha: Frontmatter `tools` parsing must support YAML block arrays, inline arrays, and comma-separated scalar strings; otherwise bracket-corrupted names such as `[read` or `ast_grep]` silently break toolbelt isolation.
- Recipe: Adopt Trellis-style fail-closed bootstrap language for OMP executor/reviewer prompts: if row id, task brief, boundary contract, curated context, or doneWhen is missing, report a blocker and do not infer the active task from repository state.
- Gotcha: Reviewer toolbelt least-privilege can conflict with review artifact requirements. If baseline reviewers are read-only, do not add broad `write`/`edit`; instead make verdict submission generate host-owned artifacts or introduce a separate narrowly scoped fix/review role.
- Recipe: Append-only evidence storage should create the header only when missing or empty, add a trailing newline if an existing file lacks one, then append exactly one RFC 4180-encoded row without parsing and rewriting historical bytes.
- Gotcha: Workflow status and row completion drift when they use different evidence sources; `getCSVWorkflowStatus()` must share the same `evidence.csv` / `.task/{rowId}.verdict.json` contract as `assertCheckPassed()` and must not count legacy `.task/{id}.json` verdict fields as current evidence.
