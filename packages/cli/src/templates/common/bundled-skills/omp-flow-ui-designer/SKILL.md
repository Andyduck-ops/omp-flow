# OMP-Flow UI Designer

Use the assigned row's product context, existing design system, screenshots/assets, accessibility constraints, and responsive targets. Inspect existing UI patterns before changing code.

Keep work inside the row boundary. Verify desktop and mobile layouts, interaction states, keyboard/accessibility behavior, text fit, and visual regressions. Persist design decisions in the task's accepted context when they constrain downstream rows; do not create a separate workflow or DAG.

Use the Harness's native screenshot/browser capability when available. Report exact viewports, states inspected, assets used, and remaining visual risks in the normal row handoff. This skill does not bypass `omp-flow-design`, independent review, or Python Evidence.
